# 🚀 DataOG (智能造数平台)

> 这是一个基于 React + FastAPI + SQLite + SQLAlchemy/DWS 构建的内网智能造数平台。集成了 DeepSeek/OpenAI 大模型智能体与本地 RAG 向量知识库，专为解决内网复杂业务场景下的数据构造难题而生。

**当前里程碑：v1.0-引入向量库** 🎉

---

## ✨ 核心特性

- **🤖 AI 协同智能造数 Agent**
  支持通过自然语言对话与 AI Agent 沟通，自动生成复杂的业务 SQL、关联测试数据，甚至根据您的意图自动理解复杂的表结构关联。
- **🧠 专属 RAG 业务知识库 (ChromaDB)**
  系统内置了轻量级向量库，AI 能够自动提取或由用户手动添加团队的特有业务知识（如：特殊字段含义、常用造数模板、状态码映射）。它会让 AI 越来越懂您的业务。
- **📊 沉浸式数据探查与 Excel 级高效录入**
  - 在线查阅 DWS/SQLite 表数据，支持双表对比与外键关系探查。
  - **类原生 Excel 体验**：表格提供直观的 `+` 号新增空行；支持直接将 Excel 或 WPS 中的多行多列数据（TSV）一键粘贴至前端网页并自动扩容表格。
  - **智能混合事务落盘**：前端会自动识别原有行（`UPDATE`）与新增行（`INSERT`），最终通过统一的批处理事务安全落盘。
- **📦 一键式极简分发部署**
  完美支持将所有 Python 运行环境（含 FastAPI 后端、SQLite/Chroma 数据库引擎）与 React 编译后的前端资源打包为单个独立的 `.exe` 文件，用 U 盘带进无外网隔离环境双击即用！

## 💻 日常开发与调试指南

如果您要在本地开发机（macOS / Windows）上修改代码，请使用以下步骤：

### 1. 启动前端服务器 (支持热更新)
新开一个终端窗口：
```bash
cd frontend
npm run dev
```
*(前端将运行在 `http://localhost:5173`)*

### 2. 启动后端 API 服务器
再新开一个终端窗口：
```bash
cd backend
source venv/bin/activate  # 激活虚拟环境 (可选)
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```
*(后端将运行在 `http://127.0.0.1:8000`)*

> **💡 开发提示**：请在浏览器访问前端地址 `http://localhost:5173`。Vite 会自动将所有 `/api` 请求代理至后端的 8000 端口，完美规避跨域问题。

## 🚀 Windows 系统下独立 EXE 打包指南 (面向交付)

如果你需要将此工具搬运至无网络的内网运行，我们提供了一键打包脚本，它会自动执行前端构建、清理缓存并调用 PyInstaller 打包。

请在一台**能连通外网的 Windows 电脑上**进行以下操作：

### 1. 基础环境准备
- 安装 [Node.js](https://nodejs.org/) (必须勾选 `Add to PATH`)。
- 安装 [Python 3.10+](https://www.python.org/downloads/) (安装界面第一页最底部，**务必勾选 `Add Python.exe to PATH`**)。
- 安装完请重启电脑或重新打开全新的命令行窗口。

### 2. 自动打包流程
打开命令行（CMD）进入项目根目录，直接运行官方包装好的脚本（在 Windows 下请使用 `py` 代替 `python`）：
```cmd
py build_exe.py
```

脚本将会自动完成：
1. 检查并提示前端是否已经构建 (若报错请进入 `frontend` 执行 `npm i && npm run build`)。
2. 安装后端 `requirements.txt` 依赖及打包工具。
3. 携带正确的 `--collect-all`、隐藏依赖项以及 `--noconsole` (无控制台) 参数执行单文件编译，并打包专属应用图标。

### 3. 提取成果！
等屏幕滚动停止并提示成功后，前往 `backend\dist` 目录下，你会看到一个名为 **`DataOG.exe`** 的单文件程序。
把它用 U 盘拷贝进内网的任意机器，双击运行即可！

## 💡 内网运行与排障说明

- **如何访问界面？** 双击 `.exe` 后，会弹出一个**可视化的控制台面板**。您可以在上面自定义端口（默认 8000），点击【启动服务】后即可打开浏览器访问 `http://127.0.0.1:端口号`。
- **如果不小心关掉了？** 如果只是关闭了浏览器，重新打开网址即可；如果想关闭服务，请点击面板上的【停止服务】或直接关闭控制台窗口。
- **AI 无法对话卡在 Thinking？** 去配置页面点击【测试连通性】。如果提示 500 报错，说明内网的 API 中转网关不支持 Agent(Tools) 解析，请联系网关管理员修复。

## 🎯 下一个里程碑: 知识管理界面架构升级 (v2.2 规划)

目前由 `<textarea>` 和基础列表手搓的“知识管理”页面在应对日益增长的业务规则时，显得简陋且难以维护。为了提供企业级的知识库管理体验，我们计划引入成熟的轻量级开源前端方案重构该模块：

### 1. 引入 TanStack Table (React Table) 重构列表
- **痛点**：目前知识列表是平铺的，没有检索、分页和字段排序功能，找知识全靠滚轮。
- **方案**：使用业界最顶级的 Headless 表格库 `TanStack Table` 构建交互式知识库大表。支持按 ID、创建时间、关键词进行**本地秒级检索、过滤与高级排序**。

### 2. 引入 Notion 风格的块级编辑器 (如 Novel.sh 或 Milkdown)
- **痛点**：原始的 `<textarea>` 无法提供语法高亮，排版极差，Markdown 编写体验不佳。
- **方案**：废弃纯文本框，整合类似 Notion 的 WYSIWYG（所见即所得）轻量级 Markdown 编辑器。支持 `/` 快捷指令呼出菜单、代码块高亮，让 Agent 提炼的业务逻辑（SQL / JSON）在界面上拥有完美的阅读体验。

### 3. 引入 Shadcn UI 或类似现代组件库
- **痛点**：现有卡片和按钮为纯手工 CSS，视觉一致性难以拓展。
- **方案**：逐步迁移至 Tailwind + `shadcn/ui`（或现有的 CSS 变量系统扩展），统一所有模态框（Dialog）、抽屉（Drawer）和气泡提示（Toast）的交互动画，使整个工具的视觉达到商业 SaaS 级水准。
